# PayloadGame HUD resource pack

A custom HUD pack that:

- makes the **boss bar invisible** (transparent sprites under
  `assets/minecraft/textures/gui/sprites/boss_bar/`), so the bar's *title* is the only
  thing drawn — turning it into a free-floating HUD canvas;
- adds the **`payloadgame:hud` font** (`assets/payloadgame/font/hud.json`):
  - `U+E010` payload-bar fill, `U+E011` payload-bar empty, `U+E002` finish flag;
  - `U+E012` health fill, `U+E013` health empty;
  - `U+E020`–`U+E025` hero portraits (enum order: Bulwark, Bullrush, Pyre, Volt, Bloom, Aegis);
  - `U+E0F1` a `space` provider glyph with **advance -1**, inserted between tiles by
    `Glyphs.seamless(...)` so the bars render flush with no visible cuts.

`Glyphs.bossBar(...)` renders the top bar (time · seamless payload bar + flag · push
count) and `Glyphs.actionBar(...)` the bottom HUD (health bar · hero portrait ·
ATTACK/DEFEND). Players who don't load the pack get a plain-text fallback (tracked by
`PackTracker` via the resource-pack status event) instead of tofu.

> The PNGs here are **functional placeholder art** generated by `generate.py`. Swap them
> for polished art (same filenames / sizes) without touching any code.

## Rebuild

```bash
python3 generate.py                 # regenerate assets into resourcepack/
cd resourcepack && zip -rq ../PayloadGameHUD.zip . -x '.*'
sha1sum ../PayloadGameHUD.zip        # put this in config.yml resource-pack.sha1
```

## Serve it

Host `PayloadGameHUD.zip` at a public URL, then in `config.yml`:

```yaml
resource-pack:
  url: "https://your.host/PayloadGameHUD.zip"
  sha1: "<sha1 from above>"
```

Players are offered the pack on join (`ResourcePackListener`). Leave `url` blank to
disable. The technique (invisible bossbar + negative-space/bitmap HUD font) follows the
approach used by packs like Modrinth's "Invisible Bars" and AmberWat's NegativeSpaceFont.
